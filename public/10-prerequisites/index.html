사전 준비

본 워크숍 환경에는 다음을 포함합니다.

개발 환경

  • 웹 브라우저를 통해 액세스 가능한 VS Code Server
  • 사전 설치된 도구 및 종속성
  • 샘플 코드 및 구성 파일
  • EKS 클러스터에 직접 액세스

Amazon EKS Auto Mode 클러스터

  • 완전 관리형 Kubernetes 환경
  • 자동 확장 및 유지보수
  • 프로덕션 준비 보안 구성
  • 통합 모니터링 및 로깅

워크숍 환경 액세스

워크숍 전 체크리스트

시작하기 전에 다음을 확인하세요:

기술적 요구사항

  • ✅ 최신 웹 브라우저 (Chrome, Firefox, Safari, Edge)
  • ✅ 안정적인 인터넷 연결
  • ✅ 웹 인터페이스에 대한 기본적인 친숙함

계정 액세스

  • ✅ Workshop Studio 액세스 (옵션 1 사용 시)
  • ✅ AWS 계정 자격 증명 (옵션 2 사용 시)
  • ✅ AWS 서비스에 대한 적절한 권한

지식 준비

  • ✅ 워크숍 소개 읽기
  • ✅ 아키텍처 개요 검토
  • ✅ 학습 목표 이해

환경 액세스 방법

워크숍 환경에 액세스하는 두 가지 옵션이 있습니다:

::::tabs

:::tab{label=“AWS Workshop Studio (권장)”} 옵션 1: AWS Workshop Studio

인프라가 미리 배포되어 준비되었습니다!

이벤트 주최자가 이미 다음을 설정했습니다:

  • ✅ 웹 브라우저를 통해 액세스 가능한 VS Code Server
  • ✅ EC2 실습 환경을 통한 docker, git 기반 실습 환경
  • ✅ 고급 실습을 위한 Auto Mode 기능을 활성화한 Amazon EKS 클러스터
  • ✅ 네트워킹 및 보안 구성

특징:

  • 관리형 워크숍 환경
  • 워크숍 후 자동 정리
  • AWS 계정 불필요
  • 간소화된 액세스 프로세스 :::

:::tab{label=“개인 AWS 계정”} 옵션 2: 개별 AWS 계정

개인 또는 회사 AWS 계정에서 이 워크숍을 독립적으로 실행하면 다음과 같은 이점이 있습니다:

장점:

  • 환경에 대한 완전한 제어
  • 워크샵 후에도 리소스를 유지할 수 있는 능력
  • 구성을 사용자 정의할 수 있는 기회
  • 인프라에 대한 더 깊은 이해

필요한 사항:

  • ✅ 적절한 권한이 있는 AWS 계정
  • ✅ 워크샵 인프라 배포
  • ✅ Docker, git 등 실습을 위한 도구 직접 설치 및 확인
  • ✅ 고급 실습을 위한 EKS 클러스터 액세스 구성

::alert[비용 경고: 이 워크샵은 AWS 컴퓨팅 인스턴스를 사용하여 요금이 발생합니다. 완료 후 리소스를 정리하는 것을 잊지 마세요!]{type=“warning”}

특징:

  • 개인 또는 회사 AWS 계정 사용
  • 워크숍 후 환경 유지
  • 리소스에 대한 완전한 제어
  • 정리 및 비용에 대한 책임 :::

::::

학습 접근법

실습 중심 철학

  • 실행을 통한 학습: 모든 개념은 실용적인 연습으로 강화됩니다
  • 실제 상황 맥락: 모든 예제는 실제 Telco 실험 및 시뮬레이션 시나리오를 사용합니다
  • 점진적 복잡성: 간단하게 시작하여 정교한 시스템으로 발전시킵니다
  • 협력적 학습: 인사이트를 공유하고 동료 참가자들을 도와주세요

지원 리소스

워크숍 중

  • 기술적 질문을 위한 워크숍 진행자
  • 참조용 포괄적인 문서
  • 일반적인 문제에 대한 문제 해결 가이드
  • 동료 지원 및 협력

워크숍 후

  • 완전한 소스 코드 저장소
  • 확장된 문서 및 튜토리얼
  • 커뮤니티 포럼 및 토론 그룹
  • 후속 리소스 및 고급 주제

성공 팁

학습 극대화

적극적 참여

  • 개념이 불분명할 때 질문하기
  • 코드 및 구성으로 실험하기
  • 다른 참가자들과 인사이트 공유하기
  • 주요 개념과 기술에 대한 메모 작성하기

워크숍을 넘어서 생각하기

  • 이러한 기술이 자신의 업무에 어떻게 적용되는지 고려하기
  • 조직에서 잠재적 사용 사례 식별하기
  • 후속 학습 및 실험 계획하기
  • 지속적인 협력을 위해 다른 참가자들과 연결하기

문제 해결 마인드셋

  • 오류는 학습 기회입니다
  • 오류 메시지를 주의 깊게 읽기
  • 문서 및 도움말 리소스 사용하기
  • 도움을 요청하는 것을 주저하지 마세요

피해야 할 일반적인 함정

단계를 서두르기

  • 각 개념을 이해하는 데 시간을 투자하세요
  • 다음 단계로 넘어가기 전에 각 단계를 확인하세요
  • 설명 섹션을 건너뛰지 마세요

모범 사례 무시하기

  • 보안 가이드라인을 따르세요
  • 적절한 명명 규칙을 사용하세요
  • 오류 처리를 구현하세요
  • 변경 사항을 문서화하세요

실험하지 않기

  • 제공된 예제의 변형을 시도해보세요
  • 다양한 구성을 테스트해보세요
  • 추가 기능을 탐색해보세요
  • 가능한 것의 경계를 넓혀보세요

시작할 준비가 되셨나요?

이제 워크숍 구조와 환경을 이해했으므로 본격적으로 워크숍에 뛰어들 준비가 되었습니다.

도움이 필요하신가요?

설정 중 문제가 발생하면:

  • 각 모듈의 문제 해결 섹션을 확인하세요
  • 워크숍 진행자에게 도움을 요청하세요
  • 동료 참가자들과 협력하세요
  • 제공된 포괄적인 문서를 참조하세요

기억하세요: 목표는 완벽함이 아니라 학습입니다. 여정을 받아들이고 최첨단 AI 시스템 구축을 즐기세요!
