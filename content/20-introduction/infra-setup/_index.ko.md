Amazon EKS는 컨테이너에 대한 대규모 스케일 확장성 및 유용한 관리 기능을 제공합니다.
여기에서는 해당 인프라에 대해 상세히 이해하고 싶은 분을 대상으로, Amazon EKS 및 Auto Mode를 중심으로 설명합니다.

::alert[모든 인프라 구성 요소가 미리 배포되고 구성되었습니다. 설정 복잡성에 대한 걱정 없이 연구 애플리케이션 학습과 실험에만 집중할 수 있습니다.]{type="success"}

## 사전 구성된 인프라 개요

워크샵 환경에는 다음 구성 요소가 포함된 완전히 작동하는 Amazon EKS 클러스터가 포함됩니다:

### 🚀 Auto Mode가 포함된 Amazon EKS 클러스터

워크샵은 컴퓨팅, 스토리지 및 네트워킹 리소스 관리의 운영 오버헤드를 제거하는 관리형 AWS Kubernetes 서비스인 **Amazon EKS Auto Mode**를 사용합니다.

#### EKS Auto Mode란?

EKS Auto Mode는 AWS가 다음을 처리하는 완전 관리형 Kubernetes 경험을 제공합니다:

- **자동 컴퓨팅 프로비저닝** - EC2 인스턴스를 수동으로 생성하거나 관리할 필요 없음
- **동적 스케일링** - 워크로드 수요에 따라 용량을 자동으로 조정
- **패치 관리** - OS 및 Kubernetes 구성 요소의 자동 업데이트
- **비용 최적화** - 지능적인 인스턴스 선택 및 Spot 인스턴스 통합

:::code{language=bash showCopyAction=true}
# EKS 클러스터 세부 정보 보기
kubectl cluster-info

# EKS Auto Mode 구성 확인
kubectl get nodes -L eks.amazonaws.com/compute-type
:::

### 💻 노드 풀 구성

클러스터에는 다양한 워크로드 유형에 최적화된 전용 노드 풀이 포함됩니다:

#### 1. **범용 노드 (멀티 아키텍처)**
- **인스턴스 유형**: 최신 세대 컴퓨팅(C), 메모리(M) 및 범용(R) 최적화 인스턴스
  - **AWS Graviton (ARM64)**: 최대 40% 더 나은 가격 대비 성능을 제공하는 ARM 기반 인스턴스
  - **x86 (AMD/Intel)**: 기존 x86 아키텍처 인스턴스
- **아키텍처 지원**: 워크로드 호환성에 따른 자동 선택이 포함된 멀티 아키텍처(amd64/arm64)
- **목적**: 일반 워크로드, 웹 서비스 및 제어 평면 구성 요소 실행
- **비용 최적화**: 
  - Graviton 인스턴스는 비교 가능한 x86 대비 최대 40% 더 나은 가격 대비 성능 제공
  - 추가 절약을 위한 Spot 및 On-Demand 용량 유형
  - Karpenter가 사용 가능한 옵션에서 가장 비용 효율적인 인스턴스 유형을 자동으로 선택
- **스케일링**: 포드 요구 사항에 따라 자동으로 스케일링하고 활용도가 낮은 노드를 통합

#### 2. **고성능 컴퓨팅 노드** 
- **인스턴스 유형**: AWS 전용 고성능 인스턴스 (c5, m5, r5 패밀리)
- **목적**: 컴퓨팅 집약적 AI/ML 워크로드 실행
- **기능**: 최적화된 네트워킹, 향상된 스토리지 성능

#### 3. **AWS Neuron 노드**
- **인스턴스 유형**: inf2, trn1 및 trn2 인스턴스
- **목적**: 대규모 언어 모델을 위한 최적화된 추론
- **이점**: 
  - AWS 전용 인스턴스 대비 추론당 최대 50% 낮은 비용
  - 최적 성능을 위한 사전 컴파일된 모델
  - 네이티브 int8 양자화 지원

:::code{language=bash showCopyAction=true}
# 사용 가능한 노드 풀 및 구성 보기
kubectl get nodepools -o wide
:::
